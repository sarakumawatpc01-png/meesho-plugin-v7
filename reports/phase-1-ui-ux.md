# Phase 1 — UI/UX Gaps Report

## Import UX
- No visible queue/progress state for long‑running imports.
- No retry/resume UI for failed imports.
- Duplicate handling is server‑side only; no bulk resolution UI.

## Blog UX
- Missing required fields in the generator UI (slug/status/tags/featured image/publish controls).
- No streaming generation feedback; only a single “generate” response.
- Lacks scheduling and revision management controls.

## Orders UX
- No bulk actions or export.
- Limited filtering (status/search only).
- No quick Meesho SKU → URL/quick‑action affordances.

## SEO UX
- Dashboard exists, but no explicit approval/rollback flow for automated changes.
- Internal linking modes are not exposed.

## Analytics UX
- Mixed configuration guidance; some tabs depend on external plugins but UI does not fully gate/guide setup.
- Widgets are not auto‑refreshing and require manual actions.

## Logging UX
- Logs table has limited filtering (type/source only).
- No severity, retention UI, or export.
